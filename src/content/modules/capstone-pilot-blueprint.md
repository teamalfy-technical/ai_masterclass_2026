objective: >-
  * define one credible AI pilot
  * narrow scope to something testable
  * name owners, controls, and metrics
  * explain why the pilot is safe enough to start and small enough to stop
coreIdea:
  - a strong pilot is narrow, owned, measurable, and reversible
  - start with one workflow, not a platform transformation
  - boundaries create traction
deliveryLead: >-
  The best first pilot is not the broadest idea. It is the clearest workflow with an owner, approval points, measurable outcomes, and a go / no-go decision.
talkingPoints:
  - named use case and real user group
  - controls, risks, and success metrics
  - 30-60-90 day next step
takeaway: >-
  A pilot should generate evidence, not theatre.
